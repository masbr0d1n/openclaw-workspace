### Duration Type Validation Errors
- **Issue**: `"Input should be a valid integer, got a number with a fractional part"` - API rejecting float durations (142.84s)
- **Root Cause**: Pydantic schema expected `int` but database and frontend sent `float`
- **Fix**: Changed all duration fields from `int` to `float` in app/schemas/playlist.py:
  - PlaylistBase.default_duration: int → float
  - PlaylistItemCreateSimple.duration: int → float
  - PlaylistUpdate.default_duration: Optional[int] → Optional[float]
  - PlaylistResponse.default_duration: int → float
  - PlaylistResponse.total_duration: int → float
  - PlaylistItemBase.duration: int → float
  - PlaylistItemResponse.duration: int → float
- **Database Schema**: Already migrated to NUMERIC(10,2) (commit 1e71c3d)
- **Commit**: a36aa4f
- **Impact**: Decimal durations now preserved correctly

### Draft Overwrite Feature
- **User Request**: "Hapus playlist draft dengan nama yang sama, timpa dengan yang paling update" (Delete draft with same name, replace with newer one)
- **Implementation**: 
  - POST endpoint: If draft name exists AND is draft, delete old and create new
  - PUT endpoint: If new name exists AND is draft, delete old and continue update
  - Published playlists still protected from duplicates (409 Conflict)
- **File**: app/api/playlists.py
- **Commits**: f709382 (schema), 5ac7372 (thumbnails), f709382 (draft overwrites)
- **Result**: Users can save drafts with same name, system auto-updates

### Thumbnail Display Issues

- **Result**: Thumbnails now show correctly when editing drafts

### Key Technical Decisions
1. **Duration Precision**: NUMERIC(10,2) allows decimal storage (e.g., 142.84s)
2. **Duration Display**: User-friendly HH:MM:SS format via formatDuration()
3. **Draft Management**: Allow duplicate names for drafts, unique for published
4. **Thumbnail Storage**: Keep in videos table, fetch via JOIN (not duplicated in playlist_items)
5. **Data Normalization**: Use videos JOIN to get name/media_type/thumbnail_data
6. **RESTful Design**: PUT for updates, POST for creates
7. **UI/UX**: Context-aware button text ("Update Draft" vs "Save Draft")
