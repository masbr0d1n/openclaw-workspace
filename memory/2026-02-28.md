### Issues Fixed:
1. **Channel ID Problem** - Frontend was sending channel_id=1 (doesn't exist)
   - Fixed: Changed to channel_id=2 (existing channel)

2. **Missing video_url in Response** - VideoResponse schema didn't include video_url
   - Fixed: Added `video_url: Optional[str] = None` to VideoResponse

3. **File Persistence** - Videos lost on container restart
   - Fixed: Added Docker volume mount `-v apistreamhub-uploads:/app/uploads`

4. **"No Video Available" Error** - Frontend couldn't find video_url in API response
   - Root cause: VideoResponse schema missing video_url field
   - Fixed: Schema update + backend rebuild

### Deployment:
- **Backend Image**: `apistreamhub-api:video-url-fix`
- **Frontend Image**: `streamhub-frontend:api-proxy-fix`
- **Backend Port**: 8001 (host) → 8000 (container)
- **Frontend Port**: 3000 (host) → 3000 (container)
- **Volume**: `apistreamhub-uploads` → `/app/uploads`

### Docker Commands:
```bash
# Backend
docker run -d --name apistreamhub-api -p 8001:8000 \
  -e DATABASE_URL="postgresql+asyncpg://postgres:postgres@host.docker.internal:5434/apistreamhub" \
  -v apistreamhub-uploads:/app/uploads \
  --add-host host.docker.internal:host-gateway \
  apistreamhub-api:video-url-fix

# Frontend
docker run -d --name streamhub-test -p 3000:3000 \
  -e BACKEND_API_URL="http://host.docker.internal:8001/api/v1" \
  --add-host host.docker.internal:host-gateway \
  streamhub-frontend:api-proxy-fix
```
